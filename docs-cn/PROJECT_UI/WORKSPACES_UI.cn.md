## 概述

Workspaces UI 提供创建、浏览和管理团队项目的协作工作空间的界面。

**布局:** MainLayout (authenticated)

## 创建 Workspace 模态框

### 触发
- 点击 "创建 Workspace" 按钮

### 表单字段

**名称** (必填)
- 输入类型: text
- 验证: 3-50 字符
- 占位符: "我的团队"

**Slug** (必填)
- 输入类型: text
- 从名称自动生成
- 可编辑
- 验证: 3-30 字符，小写，字母数字 + 连字符
- 占位符: "my-awesome-team"
- 实时可用性检查

**描述** (可选)
- 输入类型: textarea
- 验证: 最多 500 字符
- 占位符: "您的 workspace 是关于什么的？"

**类型** (必填)
- 单选按钮: 公共 / 私有
- 默认: 私有
- 帮助文本:
  - 公共: "在探索中可见，任何人都可以请求加入"
  - 私有: "仅限邀请，在公共列表中不可见"

**焦点标签** (可选)
- 输入类型: text（逗号分隔）
- 占位符: "Web 开发, Open Source, AI"
- 提交时转换为数组

### 操作
- **创建** - 提交表单
- **取消** - 关闭模态框

### 验证
- 失焦时实时验证
- 有效前提交按钮禁用
- 字段下方显示错误消息

---

## 加入 Workspace 模态框

### 触发
- 点击 "通过代码加入" 按钮

### 表单字段

**邀请码** (必填)
- 输入类型: text
- 占位符: "输入邀请码"
- 转换为大写
- 验证: 字母数字，6-20 字符

### 操作
- **加入** - 提交代码
- **取消** - 关闭模态框

### 成功
- Toast: "成功加入 workspace"
- 刷新 workspace 列表
- 关闭模态框

### 错误
- "邀请码无效或已过期"
- "邀请已达到最大使用次数"

## Workspace 详情页面

### 布局结构

```
┌─────────────────────────────────────────┐
│ Banner 图片                             │
├─────────────────────────────────────────┤
│ Workspace 名称         [编辑] [离开]    │
│ @slug • 公共 • 12 位成员                  │
│ 描述文字...                              │
│ 标签: [Web] [Open Source]               │
├─────────────────────────────────────────┤
│ 标签页: [项目] [成员] [设置]             │
├─────────────────────────────────────────┤
│ 标签页内容                               │
└─────────────────────────────────────────┘
```

### Header 部分

**Banner**
- 可选的 banner 图片
- 回退: 渐变背景

**标题区域**
- Workspace 名称 (h1)
- Slug，类型，成员数量
- 描述
- 焦点标签作为徽章

**操作** (角色相关)
- **编辑** 按钮 (所有者/管理员)
- **离开** 按钮 (除所有者外的所有成员)
- **加入** 按钮 (非成员，公共 workspaces)

### 项目标签页

**用途:** 列出 workspace 中的所有项目

**项目卡片:**
- 项目名称
- 描述
- 状态徽章 (Planning, Active 等)
- Tech stack 图标
- 成员头像
- 点击 → 导航到项目详情

**操作:**
- **创建项目** 按钮 (所有者/管理员/lead)

**空状态:**
- "还没有项目"
- "创建您的第一个项目" 按钮

### 成员标签页

**用途:** 显示和管理 workspace 成员

**成员列表:**
- 头像
- 姓名和用户名
- 角色徽章
- 标题（如果设置）
- 加入日期
- 操作菜单 (所有者/管理员):
  - 更改角色
  - 移除成员

**操作:**
- **邀请成员** 按钮 (所有者/管理员)
- **生成邀请链接** 按钮 (所有者/管理员)

**角色过滤:**
- 下拉菜单按角色过滤
- 选项: 全部, 所有者, 管理员, Lead, Developer 等

### 设置标签页

**用途:** Workspace 配置 (仅所有者/管理员)

**部分:**

1. **常规设置**
   - 编辑名称
   - 编辑描述
   - 更改类型（公共/私有）
   - 更新焦点标签

2. **邀请管理**
   - 活跃邀请列表
   - 生成新邀请
   - 撤销邀请

3. **危险区域** (仅所有者)
   - 转让所有权
   - 删除 workspace

## 编辑 Workspace 模态框

### 触发
- 点击 "编辑" 按钮 (所有者/管理员)

### 表单字段
- 与创建 Workspace 相同
- 预填充当前值

### 操作
- **保存更改** - 更新 workspace
- **取消** - 放弃更改

---

## 邀请成员模态框

### 触发
- 点击 "邀请成员" 按钮

### 选项

**1. 生成邀请链接**
- 过期时间: 下拉 (24小时, 7天, 30天, 永不过期)
- 最大使用次数: 输入 (可选，默认无限制)
- 生成按钮
- 复制链接按钮
- 分享通过: 邮箱, Telegram 等

**2. 按用户名添加**
- 用户名输入带自动完成
- 角色选择器
- 标题输入 (可选)
- 添加按钮

---

## 成员管理

### 更改角色模态框

**触发:** 点击角色徽章 → 更改角色

**表单:**
- 角色下拉菜单
- 标题输入
- 保存按钮

**验证:**
- 不能提升到等于或高于自己的角色
- 不能更改所有者角色

### 移除成员确认

**触发:** 点击操作菜单中的移除

**模态框:**
- 警告消息
- 成员姓名和角色
- 确认按钮 (破坏性)
- 取消按钮

**限制:**
- 不能移除所有者
- 不能移除等于或更高角色的用户

## API 集成

### 使用的端点
- `GET /workspaces/my` - 用户的 workspaces
- `GET /workspaces/public` - 公共 workspaces
- `GET /workspaces/:slug` - Workspace 详情
- `POST /workspaces` - 创建 workspace
- `PATCH /workspaces/:id` - 更新 workspace
- `DELETE /workspaces/:id` - 删除 workspace
- `GET /workspaces/:id/members` - 获取成员
- `POST /workspaces/:id/members` - 添加成员
- `PATCH /workspaces/:id/members/:userId` - 更新成员
- `DELETE /workspaces/:id/members/:userId` - 移除成员
- `POST /workspaces/:id/invites` - 生成邀请
- `POST /workspaces/join/:inviteCode` - 通过代码加入
