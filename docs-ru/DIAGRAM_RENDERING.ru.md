[English](../docs/DIAGRAM_RENDERING.md) | 中文 | **Русский**

**Последнее обновление:** Май 2026 г.
**Статус:** ✅ Реализовано

## Обзор

LUME поддерживает рендеринг блок-схем непосредственно в сообщениях чата с использованием синтаксиса **Mermaid**. Когда сообщение начинается с ключевого слова диаграммы, оно автоматически обнаруживается и отображается как SVG диаграмма.

---

## Backend

### Endpoint

```
POST /api/diagram/render
```

**Request:**
```json
{
  "code": "graph TD; A-->B;",
  "type": "mermaid"
}
```

**Response:**
```json
{
  "svg": "<svg>...</svg>"
}
```

### Особенности

- **Интеграция с Kroki API** - Использует `https://kroki.io/mermaid/svg` для рендеринга
- **Кеширование** - Результаты кешируются в Redis по SHA256 хешу (TTL: 1 час)
- **Валидация** - Максимальная длина кода: 50 000 символов
- **Обработка ошибок** - Возвращает 400 для некорректного кода, 502 для ошибок Kroki

## Frontend

### Автоопределение

Сообщения автоматически определяются как диаграммы, когда начинаются с:

- `graph TD` - Блок-схема сверху вниз
- `graph BT` - Блок-схема снизу вверх
- `graph LR` - Блок-схема слева направо
- `graph RL` - Блок-схема справа налево
- `flowchart TD` - Альтернативный синтаксис (сверху вниз)
- `flowchart BT` - Альтернативный синтаксис (снизу вверх)
- `flowchart LR` - Альтернативный синтаксис (слева направо)
- `flowchart RL` - Альтернативный синтаксис (справа налево)
- `pie` - Круговая диаграмма
- `gitGraph` - Граф Git

### Компонент

**Файл:** `src/components/chat/DiagramMessage.tsx`

**Особенности:**
- Скелетон загрузки во время рендеринга
- Состояние ошибки с сообщением
- Рендеринг SVG с санитизацией
- Кнопка копирования кода (показывает галочку при успехе)
- Кнопка скачивания SVG (показывает галочку при успехе)
- UI с тёмной темой и эффектом стекла

## Примеры использования

### Простая блок-схема (сверху вниз)

```
graph TD
A[Старт] --> B{Решение}
B -->|Да| C[Процесс 1]
B -->|Нет| D[Процесс 2]
```

### Блок-схема (слева направо)

```
graph LR
A[Ввод] --> B[Обработка]
B --> C[Вывод]
```

### Сложная блок-схема

```
graph TD
    subgraph Frontend["Frontend (React + TypeScript)"]
        A1[Интерфейс] --> A2[Состояние]
        A2 --> A3[API]
    end

    subgraph Backend["Backend (Node.js)"]
        B1[Express] --> B2[Маршруты]
        B2 --> B3[База данных]
    end

    A3 -->|HTTP| B2
```

### Круговая диаграмма

```
pie title Домашние животные
    "Собаки" : 386
    "Кошки" : 85
    "Птицы" : 15
```

### Git граф

```
gitGraph
    commit id: "1"
    commit id: "2"
    branch feature
    commit id: "3"
    checkout main
    commit id: "4"
```

## Интеграция

### Поток сообщений

1. Пользователь отправляет сообщение в чат
2. `detectDiagramCode()` проверяет, начинается ли сообщение с ключевого слова диаграммы
3. Если обнаружено, компонент `DiagramMessage` рендерится
4. При монтировании компонент запрашивает SVG с `/api/diagram/render`
5. SVG санитизируется и отображается
6. Чат автоматически прокручивается вниз

### Автопрокрутка

После отправки диаграммы чат автоматически прокручивается для отображения нового контента:
- Состояние: `scrollToBottomTrigger` в MessagesPage.tsx
- Триггер: после завершения `handleSendMessage()`

---

## Безопасность

- SVG санитизируется перед рендерингом (удаляются скрипты, обработчики событий)
- Максимальная длина кода ограничена (50КБ)
- Ограничение скорости через существующую защиту API
