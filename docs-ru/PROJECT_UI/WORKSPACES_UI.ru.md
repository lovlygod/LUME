## Обзор

Workspaces UI предоставляет интерфейсы для создания, просмотра и управления коллаборативными workspaces, где команды организуют свои проекты.

---

## Маршруты

### Список Workspaces
**Путь:** `/workspaces`
**Компонент:** `src/pages/workspaces/WorkspacesPage.tsx`
**Layout:** MainLayout (authenticated)

### Детали Workspace
**Путь:** `/workspaces/:slug`
**Компонент:** `src/pages/workspaces/WorkspaceDetailPage.tsx`
**Layout:** MainLayout (authenticated)

## Страница Списка Workspaces

### Структура Layout

```
┌─────────────────────────────────────┐
│ Header: "Workspaces"                │
│ [Создать Workspace] [Присоединиться]│
├─────────────────────────────────────┤
│ Вкладки: [Мои Workspaces] [Обзор]  │
├─────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐│
│ │Workspace│ │Workspace│ │Workspace││
│ │  Card   │ │  Card   │ │  Card   ││
│ └─────────┘ └─────────┘ └─────────┘│
└─────────────────────────────────────┘
```

### Вкладка "Мои Workspaces"

**Назначение:** Отображать workspaces, где пользователь является участником

**Карточка Workspace:**
- Название workspace
- Описание (сокращённое)
- Количество участников
- Бейдж роли пользователя
- Теги фокуса
- Клик → переход к деталям workspace

**Пустое состояние:**
- Сообщение: "Вы ещё не присоединились ни к одному workspace"
- CTA: Кнопка "Создайте свой первый workspace"

### Вкладка "Обзор"

**Назначение:** Просмотр публичных workspaces

**Карточка Workspace:**
- Название workspace
- Описание (сокращённое)
- Количество участников
- Теги фокуса
- Кнопка "Присоединиться" (если не участник)
- Клик → переход к деталям workspace

**Пустое состояние:**
- Сообщение: "Нет доступных публичных workspaces"

## Модальное окно "Создать Workspace"

### Триггер
- Клик по кнопке "Создать Workspace"

### Поля формы

**Название** (обязательно)
- Тип: текст
- Валидация: 3-50 символов
- Placeholder: "Моя Команда"

**Slug** (обязательно)
- Тип: текст
- Автогенерация из названия
- Редактируемый
- Валидация: 3-30 символов, строчные буквы, цифры + дефисы
- Placeholder: "moya-komanda"
- Проверка доступности в реальном времени

**Описание** (опционально)
- Тип: textarea
- Валидация: макс 500 символов
- Placeholder: "О чём ваш workspace?"

**Тип** (обязательно)
- Радио-кнопки: Публичный / Приватный
- По умолчанию: Приватный
- Подсказки:
  - Публичный: "Виден в поиске, любой может запросить присоединение"
  - Приватный: "По приглашениям, не виден в публичных списках"

**Теги фокуса** (опционально)
- Тип: текст (через запятую)
- Placeholder: "Веб-разработка, Open Source, AI"
- Преобразуется в массив при отправке

### Действия
- **Создать** - Отправить форму
- **Отмена** - Закрыть модальное окно

### Валидация
- Валидация в реальном времени при потере фокуса
- Кнопка отправки неактивна пока данные невалидны
- Сообщения об ошибках под полями

---

## Модальное окно "Присоединиться к Workspace"

### Триггер
- Клик по кнопке "Присоединиться по коду"

### Поля формы

**Код приглашения** (обязательно)
- Тип: текст
- Placeholder: "Введите код приглашения"
- Преобразование в верхний регистр
- Валидация: буквы и цифры, 6-20 символов

### Действия
- **Присоединиться** - Отправить код
- **Отмена** - Закрыть модальное окно

### Успех
- Toast: "Вы успешно присоединились к workspace"
- Обновить список workspaces
- Закрыть модальное окно

### Ошибки
- "Неверный или просроченный код приглашения"
- "Приглашение достигло максимального использования"

## Страница Деталей Workspace

### Структура Layout

```
┌─────────────────────────────────────────┐
│ Banner Image                            │
├─────────────────────────────────────────┤
│ Workspace Name          [Редактировать] │
│ @slug • Публичный • 12 участников        │
│ Описание...                              │
│ Теги: [Веб] [Open Source]               │
├─────────────────────────────────────────┤
│ Вкладки: [Проекты] [Участники] [Настройки]│
├─────────────────────────────────────────┤
│ Контент вкладки                          │
└─────────────────────────────────────────┘
```

### Секция Header

**Banner**
- Опциональное изображение баннера
- Fallback: градиентный фон

**Область заголовка**
- Название workspace (h1)
- Slug, тип, количество участников
- Описание
- Теги фокуса как бейджи

**Действия** (зависят от роли)
- Кнопка **Редактировать** (владелец/админ)
- Кнопка **Покинуть** (все участники кроме владельца)
- Кнопка **Присоединиться** (не-участники, публичные workspaces)

### Вкладка "Проекты"

**Назначение:** Список всех проектов в workspace

**Карточка проекта:**
- Название проекта
- Описание
- Бейдж статуса (Planning, Active и т.д.)
- Иконки tech stack
- Аватары участников
- Клик → переход к деталям проекта

**Действия:**
- Кнопка **Создать проект** (владелец/админ/lead)

**Пустое состояние:**
- "Пока нет проектов"
- Кнопка "Создайте свой первый проект"

### Вкладка "Участники"

**Назначение:** Отображение и управление участниками workspace

**Список участников:**
- Аватар
- Имя и username
- Бейдж роли
- Название (если задано)
- Дата присоединения
- Меню действий (владелец/админ):
  - Изменить роль
  - Удалить участника

**Действия:**
- Кнопка **Пригласить участников** (владелец/админ)
- Кнопка **Создать пригласительную ссылку** (владелец/админ)

**Фильтр по роли:**
- Dropdown для фильтрации по роли
- Опции: Все, Владелец, Админ, Lead, Developer и т.д.

### Вкладка "Настройки"

**Назначение:** Конфигурация workspace (только владелец/админ)

**Секции:**

1. **Общие настройки**
   - Редактировать название
   - Редактировать описание
   - Изменить тип (публичный/приватный)
   - Обновить теги фокуса

2. **Управление приглашениями**
   - Список активных приглашений
   - Создать новое приглашение
   - Отозвать приглашение

3. **Опасная зона** (только владелец)
   - Передать право собственности
   - Удалить workspace

## Модальное окно "Редактировать Workspace"

### Триггер
- Клик по кнопке "Редактировать" (владелец/админ)

### Поля формы
- Те же что и при создании Workspace
- Заполнены текущими значениями

### Действия
- **Сохранить изменения** - Обновить workspace
- **Отмена** - Отменить изменения

---

## Модальное окно "Пригласить участников"

### Триггер
- Клик по кнопке "Пригласить участников"

### Опции

**1. Создать пригласительную ссылку**
- Срок действия: dropdown (24ч, 7 дней, 30 дней, Никогда)
- Максимум использований: input (опционально, по умолчанию без лимита)
- Кнопка создать
- Кнопка копировать ссылку
- Поделиться через: Email, Telegram и т.д.

**2. Добавить по username**
- Input username с автодополнением
- Селектор роли
- Input названия (опционально)
- Кнопка добавить

---

## Управление участниками

### Модальное окно "Изменить роль"

**Триггер:** Клик по бейджу роли → Изменить роль

**Форма:**
- Dropdown роли
- Input названия
- Кнопка сохранить

**Валидация:**
- Нельзя повысить до равной или более высокой роли чем у себя
- Нельзя изменить роль владельца

### Подтверждение удаления участника

**Триггер:** Клик "Удалить" в меню действий

**Модальное окно:**
- Предупреждающее сообщение
- Имя участника и роль
- Кнопка подтверждения (деструктивная)
- Кнопка отмены

**Ограничения:**
- Нельзя удалить владельца
- Нельзя удалить пользователей с равной или более высокой ролью

## Интеграция с API

### Используемые Endpoints
- `GET /workspaces/my` - Workspaces пользователя
- `GET /workspaces/public` - Публичные workspaces
- `GET /workspaces/:slug` - Детали workspace
- `POST /workspaces` - Создать workspace
- `PATCH /workspaces/:id` - Обновить workspace
- `DELETE /workspaces/:id` - Удалить workspace
- `GET /workspaces/:id/members` - Получить участников
- `POST /workspaces/:id/members` - Добавить участника
- `PATCH /workspaces/:id/members/:userId` - Обновить участника
- `DELETE /workspaces/:id/members/:userId` - Удалить участника
- `POST /workspaces/:id/invites` - Создать приглашение
- `POST /workspaces/join/:inviteCode` - Присоединиться по коду
