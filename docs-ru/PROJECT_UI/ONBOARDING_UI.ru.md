## Обзор

Onboarding UI направляет новых пользователей через 4-шаговый мастер для настройки профиля, навыков, целей и предпочтений workspace.

---

## Маршрут

**Путь:** `/onboarding`

**Компонент:** `src/pages/onboarding/OnboardingPage.tsx`

**Layout:** MainLayout (authenticated)

**Доступ:** Защищённый маршрут (требуется авторизация)

---

## Пользовательский поток

```
Шаг 1: Профиль → Шаг 2: Навыки → Шаг 3: Цели → Шаг 4: Workspace → Завершение → Редирект на /feed
```

---

## Шаг 1: Настройка профиля

### Назначение
Определить основную роль пользователя в экосистеме разработчиков.

### UI элементы
- **Заголовок:** "Какова ваша основная роль?"
- **Селектор роли:** Сетка карточек ролей
- **Навигация:** Кнопка "Далее" (неактивна пока не выбрано)

### Опции ролей
- Разработчик
- Frontend разработчик
- Backend разработчик
- Fullstack разработчик
- UI/UX дизайнер
- Telegram Bot разработчик
- Game разработчик
- Основатель
- Студент
- Open Source контрибьютор
- Другое

### Взаимодействие
1. Пользователь кликает карточку роли
2. Карточка подсвечивается акцентным цветом
3. Кнопка "Далее" становится активной
4. Клик "Далее" → переход к Шагу 2

---

## Шаг 2: Выбор навыков

### Назначение
Захватить технические навыки и экспертизу пользователя.

### UI элементы
- **Заголовок:** "Какие у вас навыки?"
- **Категории навыков:** Раскрывающиеся секции
- **Теги навыков:** Мультивыбор чипов
- **Выбранные навыки:** Область отображения выбранных навыков
- **Навигация:** Кнопка "Назад", кнопка "Далее"

### Категории навыков
- **Frontend:** React, Next.js, Vue, Tailwind, TypeScript
- **Backend:** Node.js, Python, FastAPI, Django, NestJS, Express
- **Bots:** Aiogram, Telethon, Telegraf, Telegram Mini Apps
- **Базы данных:** PostgreSQL, SQLite, MongoDB, Redis
- **Дизайн:** Figma, UI Design, UX Design, Branding
- **Другое:** Electron, C#, WPF, Unity, Godot, Rust, Go

### Взаимодействие
1. Пользователь кликает чипы навыков для выбора/отмены
2. Выбранные навыки появляются в области сводки
3. Можно выбрать несколько навыков из разных категорий
4. Клик "Далее" → переход к Шагу 3

---

## Шаг 3: Определение целей

### Назначение
Понять, чего пользователь хочет достичь на платформе.

### UI элементы
- **Заголовок:** "Каковы ваши цели?"
- **Карточки целей:** Мультивыбор карточек с иконками
- **Навигация:** Кнопка "Назад", кнопка "Далее"

### Опции целей
- Найти команду
- Найти проект
- Показать мой проект
- Найти разработчика
- Общаться с indie разработчиками
- Собрать свою команду
- Создать open-source проект
- Найти фриланс/работу
- Просто просматривать проекты

### Взаимодействие
1. Пользователь кликает карточки целей для выбора/отмены
2. Выбранные карточки подсвечиваются акцентным цветом
3. Можно выбрать несколько целей
4. Клик "Далее" → переход к Шагу 4

---

## Шаг 4: Настройка Workspace

### Назначение
Подключить пользователя к workspace или позволить создать его.

### UI элементы
- **Заголовок:** "Присоединяйтесь или создайте workspace"
- **Три опции:**
  1. **Создать новый Workspace**
     - Input названия
     - Input slug (автогенерация из названия)
     - Textarea описания
     - Селектор типа (Публичный/Приватный)
     - Input тегов фокуса
  2. **Присоединиться к существующему Workspace**
     - Input кода приглашения
     - Кнопка "Присоединиться"
  3. **Пропустить сейчас**
     - Кнопка "Пропустить"

### Форма создания Workspace
- **Название:** Обязательно, 3-50 символов
- **Slug:** Автогенерация, редактируемый, 3-30 символов
- **Описание:** Опционально, макс 500 символов
- **Тип:** Публичный или Приватный radio buttons
- **Теги фокуса:** Через запятую

### Форма присоединения к Workspace
- **Код приглашения:** Обязательно, буквенно-цифровой код
- **Валидация:** Проверка валидности кода в реальном времени

### Взаимодействие
1. Пользователь выбирает одну из трёх опций
2. Если Создать: заполняет форму и кликает "Создать"
3. Если Присоединиться: вводит код и кликает "Присоединиться"
4. Если Пропустить: кликает "Пропустить"
5. При успехе → переход к завершению

---

## Завершение

### Действия
1. Отметить onboarding как завершённый в базе данных
2. Очистить localStorage draft
3. Показать сообщение об успехе
4. Редирект на `/feed`

### Сообщение об успехе
- Toast уведомление: "Добро пожаловать в LUME! Ваш профиль настроен."

---

## Сохранение черновика

### Ключ LocalStorage
`lume_onboarding_draft_v1`

### Сохранённые данные
```json
{
  "step": 2,
  "primaryRole": "Frontend разработчик",
  "skills": ["React", "TypeScript"],
  "goals": ["Найти команду"],
  "workspace": {
    "name": "Моя Команда",
    "slug": "moya-komanda",
    "description": "",
    "type": "private",
    "focusTags": ""
  }
}
```

### Поведение
- Сохраняется при каждой смене шага
- Восстанавливается при загрузке страницы
- Очищается при завершении или ручной очистке

## Анимации

### Переходы страниц
- **Вход:** Slide in справа с fade
- **Выход:** Slide out влево с fade
- **Длительность:** 300ms
- **Easing:** ease-in-out

### Индикатор шага
- Progress bar показывающий текущий шаг (1/4, 2/4 и т.д.)
- Анимированный переход ширины

---

## Адаптивный дизайн

### Desktop (≥1024px)
- Центрированный layout карточки
- Макс ширина: 800px
- Двухколоночная сетка для карточек ролей/целей

### Tablet (768px - 1023px)
- Карточка на всю ширину
- Одноколоночная сетка

### Mobile (<768px)
- Layout на весь экран
- Стэк элементов
- Увеличенные touch targets

---

## Обработка ошибок

### Ошибки валидации
- Встроенные сообщения об ошибках под инпутами
- Красная граница на невалидных полях
- Предотвращение продолжения пока не исправлено

### Ошибки API
- Toast уведомление с сообщением об ошибке
- Кнопка повтора
- Форма остаётся заполненной для повтора

### Сетевые ошибки
- Сообщение "Соединение потеряно"
- Автоповтор при переподключении
- Черновик сохранён для восстановления

## Доступность

### Навигация клавиатуры
- Tab через все интерактивные элементы
- Enter для выбора/отправки
- Escape для возврата

### Скрин ридеры
- ARIA лейблы на всех инпутах
- Объявления ролей
- Объявления прогресса

### Управление фокусом
- Видимые индикаторы фокуса
- Focus trap внутри модального окна
- Автофокус на первом инпуте

## Используемые компоненты

### UI компоненты
- `CustomSelect` - Dropdown селекторы
- `Textarea` - Multiline text input
- `Button` - Primary/secondary действия
- `Card` - Контейнер для шагов
- `Badge` - Чипы навыков/тегов
- `Input` - Текстовые инпуты

### Утилиты
- `slugify()` - Конвертация названия в URL-безопасный slug
- `toast` - Система уведомлений
- `AnimatePresence` - Framer Motion анимации

---

## Интеграция с API

### Используемые Endpoints
- `GET /onboarding/status` - Проверить статус завершения
- `POST /onboarding/profile` - Сохранить роль
- `POST /onboarding/skills` - Сохранить навыки
- `POST /onboarding/goals` - Сохранить цели
- `POST /onboarding/workspace` - Создать/присоединиться к workspace
- `POST /onboarding/complete` - Отметить как завершённый
