## Обзор

Projects UI предоставляет интерфейсы для создания, просмотра и управления проектами. Проекты могут быть независимыми или связаны с workspaces, с управлением задачами, командной коллаборацией и функциями демонстрации.

---

## Маршруты

### Список Проектов
**Путь:** `/projects`
**Компонент:** `src/pages/projects/ProjectsPage.tsx`
**Layout:** MainLayout (authenticated)

### Детали Проекта
**Путь:** `/projects/:slug`
**Компонент:** `src/pages/projects/ProjectDetailPage.tsx`
**Layout:** MainLayout (authenticated)

## Страница Списка Проектов

### Структура Layout

```
┌─────────────────────────────────────┐
│ Header: "Projects"                  │
│ [Создать проект]                    │
├─────────────────────────────────────┤
│ Вкладки: [Мои проекты] [Обзор]     │
├─────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐│
│ │ Проект  │ │ Проект  │ │ Проект  ││
│ │  Card   │ │  Card   │ │  Card   ││
│ └─────────┘ └─────────┘ └─────────┘│
└─────────────────────────────────────┘
```

### Вкладка "Мои проекты"

**Назначение:** Отображение проектов пользователя

**Карточка проекта:**
- Banner/логотип изображение
- Название проекта
- Описание (сокращённое)
- Бейдж статуса (Idea, Building, Testing, Launched, Paused, Archived)
- Иконки tech stack (макс 5, +N)
- Аватары участников (макс 3, +N)
- Прогресс задач (X/Y выполнено)
- Бейдж роли пользователя
- Клик → переход к деталям проекта

**Пустое состояние:**
- Сообщение: "Вы ещё не создали ни одного проекта"
- CTA: Кнопка "Создайте свой первый проект"

### Вкладка "Обзор"

**Назначение:** Просмотр публичных проектов

**Фильтры:**
- Dropdown статуса (Все, Idea, Building, Testing, Launched, Paused, Archived)
- Фильтр по tech stack (мультивыбор)
- Фильтр по тегам
- Toggle "Open source"

**Карточка проекта:**
- Та же что и в "Мои проекты"
- Бейдж "Ищу участников" (если применимо)
- Бейдж "Open Source" (если применимо)
- Количество звёзд на GitHub (если связан)

**Пустое состояние:**
- Сообщение: "Публичные проекты не найдены"
- Предложение изменить фильтры

## Модальное окно "Создать проект"

### Триггер
- Клик по кнопке "Создать проект"

### Секции формы

#### Базовая информация

**Название** (обязательно)
- Тип: текст
- Валидация: 3-100 символов
- Placeholder: "Мой крутой проект"

**Slug** (обязательно)
- Тип: текст
- Автогенерация из названия
- Редактируемый
- Валидация: 3-50 символов, строчные буквы, цифры + дефисы
- Проверка доступности в реальном времени

**Описание** (опционально)
- Textarea
- Валидация: макс 1000 символов
- Поддержка Markdown
- Toggle предпросмотра

#### Детали проекта

**Статус** (обязательно)
- Dropdown: Idea, Building, Testing, Launched, Paused, Archived
- По умолчанию: Idea

**Видимость** (обязательно)
- Радио: Публичный / Приватный
- По умолчанию: Приватный

**Tech Stack** (опционально)
- Мультивыбор с автодополнением
- Популярные опции: React, Node.js, Python и т.д.
- Разрешено кастомный ввод
- Макс 20 элементов

**Теги** (опционально)
- Input через запятую
- Преобразуется в чипы
- Макс 10 тегов

#### Ссылки

**GitHub URL** (опционально)
- Тип: url
- Валидация: валидный GitHub URL
- Автополучение инфо о репозитории (звёзды, язык)

**Demo URL** (опционально)
- Тип: url
- Валидация: валидный URL
- Автопрефикс https:// если отсутствует

#### Настройки

**Ищу участников** (опционально)
- Checkbox
- Отображается в обзоре с бейджем

**Open Source** (опционально)
- Checkbox
- Добавляет бейдж open source

### Действия
- **Создать проект** - Отправить форму
- **Отмена** - Закрыть модальное окно

### Валидация
- Валидация в реальном времени
- Кнопка отправки неактивна пока невалидно
- Сообщения об ошибках встроенные

## Страница Деталей Проекта

### Структура Layout

```
┌─────────────────────────────────────────┐
│ Banner Image                            │
├─────────────────────────────────────────┤
│ Logo  Название проекта    [Настройки]   │
│       @slug • Статус • Видимость         │
│       Описание...                        │
│       Stack: [React] [Node] [Postgres]  │
│       Теги: [Веб] [SaaS]                │
│       [GitHub] [Demo] [Ищу команду]      │
├─────────────────────────────────────────┤
│ Вкладки: [Обзор] [Задачи] [Команда] [Чат]│
├─────────────────────────────────────────┤
│ Контент вкладки                          │
└─────────────────────────────────────────┘
```

### Секция Header

**Banner**
- Полноширинное изображение баннера
- Fallback: градиент

**Logo и заголовок**
- Логотип проекта (слева)
- Название проекта (h1)
- Slug, статус, видимость
- Описание (раскрываемое если длинное)
- Бейджи tech stack
- Теги
- Кнопки действий

**Ссылки**
- Кнопка GitHub (с количеством звёзд)
- Кнопка Demo
- Бейдж "Ищу участников"

**Действия** (зависят от роли)
- Кнопка **Настройки** (admin/lead/manager)
- Кнопка **Покинуть** (участники)
- Кнопка **Присоединиться** (не-участники, если ищет участников)

### Вкладка "Обзор"

**Назначение:** Сводка проекта и активность

**Секции:**

1. **О проекте**
   - Полное описание (рендер Markdown)
   - Дата создания
   - Последнее обновление

2. **Быстрая статистика**
   - Всего задач
   - Выполненных задач
   - Участников команды
   - Звёзды на GitHub (если связан)

3. **Недавняя активность**
   - Обновления задач
   - Новые участники
   - Изменения статуса
   - Лимит: 10 элементов

4. **Избранное команды**
   - Аватары участников с ролями
   - Ссылка "Показать всех" на вкладку Команда

### Вкладка "Задачи"

**Назначение:** Kanban доска для управления задачами

**Layout:**
```
┌──────────┬──────────┬──────────┬──────────┐
│   Todo   │ Building │ Testing  │  Done    │
├──────────┼──────────┼──────────┼──────────┤
│ [Task 1] │ [Task 3] │ [Task 5] │ [Task 7] │
│ [Task 2] │ [Task 4] │ [Task 6] │ [Task 8] │
│    +     │    +     │    +     │    +     │
└──────────┴──────────┴──────────┴──────────┘
```

**Карточка задачи:**
- Название
- Бейдж приоритета (цветокодированный)
- Аватар исполнителя
- Количество комментариев
- Дата создания
- Drag для перемещения между колонками

**Действия:**
- Кнопка **Создать задачу** (admin/lead/manager/owner)
- Dropdown **Фильтр** (по исполнителю, приоритету)
- Dropdown **Сортировка** (по дате, приоритету)

**Модальное окно деталей задачи:**
- Полное описание
- Dropdown статуса
- Селектор приоритета
- Селектор исполнителя
- Тред комментариев
- Кнопки редактирования/удаления (если разрешено)

### Вкладка "Команда"

**Назначение:** Управление командой проекта

**Список участников:**
- Аватар
- Имя и username
- Бейдж роли
- Дата присоединения
- Меню действий (admin/lead/manager):
  - Изменить роль
  - Удалить участника

**Действия:**
- Кнопка **Пригласить** (admin/lead/manager)
- Кнопка **Сгенерировать ссылку** (admin/lead/manager)

**Фильтр по роли:**
- Dropdown: Все, Owner, Admin, Lead, Manager, Developer и т.д.

### Вкладка "Чат"

**Назначение:** Командный чат проекта

**Отображается:** Связанный групповой чат

**Если чат не связан:**
- Сообщение "Чат не связан"
- Кнопка "Связать чат" (только владелец)

**Если чат связан:**
- Встроенный интерфейс чата
- Сообщения группы/канала

## Модальное окно настроек проекта

### Триггер
- Клик по кнопке "Настройки" в header проекта

### Вкладки настроек

#### Общие
- Загрузка логотипа
- Название
- Описание
- Статус
- Tech Stack
- GitHub URL
- Demo URL
- Ищу участников
- Open Source

#### Участники
- Поиск пользователей по username
- Добавление по ID
- Изменение ролей
- Удаление участников

#### Чат
- Связать чат с проектом (выбор из существующих)
- Отвязать чат

#### Опасная зона (только владелец)
- Удалить проект

### Действия
- **Сохранить изменения**
- **Отмена**

## Модальное окно "Создать задачу"

### Триггер
- Клик "+" в колонке задач или кнопка "Создать задачу"

### Поля формы

**Название** (обязательно)
- Input: текст
- Валидация: 3-200 символов

**Описание** (опционально)
- Textarea
- Поддержка Markdown
- Макс 2000 символов

**Статус** (обязательно)
- Dropdown: Todo, In Progress, Review, Done
- По умолчанию: на основе кликнутой колонки

**Приоритет** (обязательно)
- Dropdown: Low, Medium, High, Urgent
- По умолчанию: Medium
- Цветокодированный

**Исполнитель** (опционально)
- Dropdown: участники проекта
- С поиском
- Превью аватара

### Действия
- **Создать задачу**
- **Отмена**

## Адаптивный дизайн

### Desktop (≥1024px)
- Трёхколоночная сетка для карточек проектов

- Двухколоночная сетка

- Одноколоночный список
- Вертикальный Kanban (стек колонок)
- Модальные окна в виде нижнего листа
- Swipe жесты для статуса задачи

## Интеграция с API

### Используемые Endpoints
- `GET /projects/my` - Проекты пользователя
- `GET /projects/public` - Публичные проекты
- `GET /projects/:slug` - Детали проекта
- `POST /projects` - Создать проект
- `PATCH /projects/:id` - Обновить проект
- `POST /projects/:id/logo` - Загрузить логотип
- `DELETE /projects/:id` - Удалить проект
- `GET /projects/:id/members` - Получить участников
- `POST /projects/:id/members` - Добавить участника
- `DELETE /projects/:id/members/:userId` - Удалить участника
- `POST /projects/:id/leave` - Покинуть проект
- `POST /projects/:id/invite` - Создать приглашение
- `POST /projects/:id/join` - Присоединиться к проекту
- `POST /projects/:id/chat` - Связать чат
- `DELETE /projects/:id/chat` - Отвязать чат
- `GET /projects/:id/search-users` - Поиск пользователей
- `GET /projects/:projectId/tasks` - Получить задачи
- `POST /projects/:projectId/tasks` - Создать задачу
- `PATCH /tasks/:taskId` - Обновить задачу
- `DELETE /tasks/:taskId` - Удалить задачу
- `POST /tasks/:taskId/comments` - Добавить комментарий
